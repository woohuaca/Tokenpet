# 阿在宠物动作、表情与声音设计稿

## 这份设计稿要解决什么问题

动作、表情和声音不是“让桌宠更热闹”的装饰层，而是宠物最核心的情绪语言。

如果这层做不好，用户看到的就只是：

- 一个会记账的插件
- 一个会播状态的窗口
- 一个被功能驱动的小动画

如果这层做好了，用户感受到的才会是：

- 它在等我
- 它刚刚真的被我拍对了
- 它今天有点别扭
- 这只的脾气和别的那只不一样

因此，这份稿子的目标不是列更多动作，而是定义：

1. 哪些情绪必须被清楚地表现出来
2. 这些情绪该用怎样的动作、表情和声音表达
3. 不同成长偏向下，同一种情绪应如何分化
4. V1 应先做哪些，后做哪些

## 设计结论

阿在的动作系统应先围绕“关系时刻”设计，而不是围绕“功能操作”设计。

最重要的不是它会不会跳、会不会走，而是它在下面这些时刻能不能准确打动人：

- 等你回应
- 被你拍对
- 被你拍偏
- 没等到你
- 自己消化情绪
- 吃到高质量粮
- 仓压过高时的胀与闷

所以后面的动作设计顺序必须是：

1. 先定义情绪谱系
2. 再定义动作节奏
3. 再定义眼、耳、尾、身体的表情语法
4. 最后再给关键节点补声音

## 第一原则：动作先表达情绪，再表达功能

动作设计的第一问题不是“它现在在做什么事”，而是“它现在在感受什么”。

同样是等待：

- 普通等待只是站住
- 有情绪的等待会带着期待、试探、克制或发劲

同样是整理：

- 功能性整理只是清洁动作
- 有情绪的整理会表现出焦躁、自我安抚、胀仓后的顺毛感

因此后续所有动作都要先标明所属情绪，而不是先标明所属功能。

## 第二原则：表情应极少，但必须极准

阿在不适合做成“几十套夸张脸谱”的卡通型宠物。

更合适的方向是：

- 眼睛开合幅度
- 眼神聚焦与否
- 嘴角上扬、抿住或轻张
- 耳朵立起、后缩或下落
- 尾巴松开、卷起、炸开或轻扫

也就是说，表情靠微小变化形成情绪，而不是靠复杂表情包堆砌。

这样更符合现在的产品方向：

- 极简
- 高情绪价值
- 能长期陪伴而不显得吵

## 第三原则：声音必须短、轻、可关闭

声音值得做，但不能做成“桌面上一直叫的宠物”。

阿在的声音更适合是带触感的短声音，而不是旋律型或语言型音频。

推荐方向：

- 拍对时：短促满足的呼噜感、啵感
- 想要你时：轻轻试探的鼻音、蹭感
- 拍偏时：顿一下的卡壳音
- 整理自己时：很轻的抖毛、舔毛细碎声
- 吃到高质量粮时：亮一点的小庆祝音

声音设计原则：

- 单次极短
- 不频繁触发
- 默认低侵扰
- 必须能关闭
- 重点是“身体感”，不是“提示感”

## 当前形象体系

### 结论

现在吞金兽已经有 `6 个主形象类别`，并且叠加了阶段、体态、配件、耳尾纹理与气场组合。

但从用户感知角度看，真正足够强的“第一眼差异”还主要集中在主形态上。下一版不该继续追求组合数，而要优先补强“强辨识的大类轮廓”。

### 当前 6 大主形象

| 主形象 | 当前气质 | 现在最容易被看到的特征 | 还不够强的地方 |
| --- | --- | --- | --- |
| 中性 | 原生、过渡、未定型 | 颜色中性，整体最基础 | 记忆点偏弱，容易像默认款 |
| 建造型 | 能干、动手、偏执行 | 更偏利落，有工具感挂件 | 身体轮廓还不够“工程感” |
| 学者型 | 安静、专注、偏思考 | 眼镜、卷轴、偏文气 | 轮廓与动作还不够“脑力型” |
| 指挥型 | 控制、秩序、带决断 | 额带、胸牌、站姿更稳 | 还缺更强的压场感 |
| 社交型 | 亲近、温暖、会贴人 | 更柔软，更容易显得可爱 | 还缺明显的“主动靠近感” |
| 混乱型 | 毛边、躁动、带故障感 | 电尾、碎光、躁气偏强 | 还缺真正危险又好玩的边缘感 |

### 当前形象系统实际由 4 层组成

1. `成长阶段`
   现在有宠物蛋、发芽期、幼年期、成长期、成熟体、传说体 6 段。
2. `主形态`
   现在有中性、建造型、学者型、指挥型、社交型、混乱型 6 类。
3. `身份体态与配件`
   现在有原生型、绒软型、利落型、探路型、安定型几类轮廓倾向，并带围巾、腮红、额带、胸牌、轨道、披肩等可见配件。
4. `生成特征`
   现在耳、尾、纹、气场各有 4 组，会组合出大量变化。

### 当前判断

从“代码组合数”看，现在外观变化已经很多。

但从“用户能不能一眼记住”看，现在还更像：

- 同一只基础团子，长出很多小差异

而还不够像：

- 明显不同脾气、不同体质、不同存在感的几类吞金兽

所以接下来最重要的，不是继续加更多耳朵、尾巴、纹理，而是优先把大类轮廓拉得更开。

## 下一版最该补的 3 种强辨识方向

### 结论

下一版最值得补的，不是新增 3 个随机小配件，而是补 3 种用户一眼能认出的“强角色型”。

建议优先补下面 3 种：

1. `仓鼠仓压型`
2. `小龙猛长型`
3. `贴贴精灵型`

这 3 种正好对应当前最有产品价值的三类体验：

- 吞很多后的胀仓感
- 高能高成长时的爆发感
- 高关系、高情绪价值时的亲近感

### 1. 仓鼠仓压型

要解决的问题：

- 现在“压仓”更多是状态变化，还不够像一种鲜明物种感

建议形象方向：

- 脸颊更鼓
- 身体更短圆
- 前爪更像抱仓
- 肚皮和仓位更有“塞满了”的视觉语言

建议互动方向：

- 整理粮仓
- 抖一抖再压回去
- 被拍开时像把闷着的粮劲吐出来

为什么优先：

- 这是吞金兽最独特的一类产品语言，最容易把 `Token -> 吞食 -> 仓压` 做成记忆点

### 2. 小龙猛长型

要解决的问题：

- 现在“猛长”已经有动效，但还不够像一种真正会冲、会窜、会发亮的成长体

建议形象方向：

- 轮廓更修长
- 耳尾更尖锐或更像角翼
- 身上的光轨更强
- 重心更前，更有准备冲出去的感觉

建议互动方向：

- 拍准后有明显余波
- 跑动、回弹、甩尾幅度更大
- 高能状态下有更强的蓄力感

为什么优先：

- 这是最适合承接“高质量 Token 喂养后明显长起来”的爽感形象

### 3. 贴贴精灵型

要解决的问题：

- 现在“社交型”和“细养型”虽然温柔，但还没有一类特别强的情绪价值代表形象

建议形象方向：

- 轮廓更软
- 脸更开
- 眼神更圆
- 靠近用户时有更明显的贴边、蹭手、回头看人的姿态

建议互动方向：

- 等待时不是站住，而是轻轻贴边
- 拍准后不是泛用开心，而是明显黏过来
- 错过后不是单纯缩回，而是带一点安静失落

为什么优先：

- 这是最能直接拉高“情绪价值”的一类，也是最容易让用户愿意每天看它一眼的形象

## 下一版形象设计取舍原则

### 要做的

- 优先加强大轮廓差异
- 优先让不同状态有不同身体重心
- 优先让互动结果在体态上可见

### 不急着做的

- 不急着继续加很多小挂件
- 不急着继续扩很多随机纹理
- 不急着把每一类都做成复杂美术设定

### 判断标准

如果用户在 1 秒内看一眼，就能回答下面至少两个问题，说明形象方向是对的：

1. 它今天大概是什么脾气
2. 它现在想不想跟我互动
3. 它是偏胀仓、偏猛长，还是偏亲近
4. 这只和上一只是不是明显不是同一类

## 三种新形象的实现稿

### 结论

这 3 种形象不能只停留在“概念词”，而要直接映射到可以开发的 4 个层面：

1. 大轮廓
2. 局部器官
3. 默认动作重心
4. 互动后的余波

只有这 4 层一起改，用户才会觉得它真的变成了另一类吞金兽。

### 1. 仓鼠仓压型实现稿

| 维度 | 实现要求 |
| --- | --- |
| 大轮廓 | 身体更短、更圆、更贴地，整体像把粮含在肚里 |
| 局部器官 | 脸颊鼓一点，前爪更短更靠前，腹部更明显 |
| 默认重心 | 重心偏下，站姿更稳，像在护着肚里的东西 |
| 待机动作 | 小幅抖毛、整理腹前、原地压一压粮仓 |
| 等待动作 | 不是往前冲，而是先守着仓、再抬头看你 |
| 拍准反馈 | 像把闷着的仓气拍松，身体先一鼓再一泄 |
| 拍偏反馈 | 只松开一点点，又马上重新把粮压回去 |
| 错过反馈 | 身体缩得更紧，像把没消化完的东西护住 |
| 动效关键词 | 鼓、压、护、泄、抖 |

建议第一版实现顺序：

1. 先改 body width / height bias
2. 再加脸颊和腹部轮廓
3. 最后加“压仓整理”的待机动作

### 2. 小龙猛长型实现稿

| 维度 | 实现要求 |
| --- | --- |
| 大轮廓 | 身体更修长，头尾拉开，整体更像会窜出去 |
| 局部器官 | 耳更尖，尾更长，允许出现角感或翼感的局部轮廓 |
| 默认重心 | 重心前倾，像一直在蓄力 |
| 待机动作 | 不是安静站着，而是轻微前探、摆尾、回弹 |
| 跑动动作 | 步幅更大，尾巴甩动更明显，速度线更利落 |
| 拍准反馈 | 会有明显余波，像能量被拍亮后往外一圈圈散开 |
| 拍偏反馈 | 身体先被带动，但尾和头不同步，像劲没完全顺开 |
| 错过反馈 | 不会立刻委屈，而是先多抖一截躁劲 |
| 动效关键词 | 冲、亮、弹、甩、窜 |

建议第一版实现顺序：

1. 先改轮廓和尾摆幅度
2. 再加强跑动与等待的体态差
3. 最后做拍准后的余波光效

### 3. 贴贴精灵型实现稿

| 维度 | 实现要求 |
| --- | --- |
| 大轮廓 | 轮廓更软，脸更开，整体像更容易靠近人 |
| 局部器官 | 眼更圆，嘴角更松，耳朵更容易前探 |
| 默认重心 | 不是守中，而是微微朝用户方向偏移 |
| 待机动作 | 轻贴边、回头看、慢慢蹭近一点 |
| 等待动作 | 不只停住，而是带着试探感地轻轻靠近 |
| 拍准反馈 | 明显贴过来、呼噜、回蹭，有被接住的感觉 |
| 拍偏反馈 | 不是生硬卡住，而是歪头再看你一眼 |
| 错过反馈 | 缩回去但不炸毛，更像安静失落 |
| 动效关键词 | 贴、蹭、靠、回头、呼噜 |

建议第一版实现顺序：

1. 先改眼口和身体前倾
2. 再强化贴边等待动作
3. 最后做拍准后的黏人余波

## 开发优先级

### P0：先做“看一眼就知道不是同一种”

- 仓鼠仓压型的大轮廓
- 小龙猛长型的大轮廓
- 贴贴精灵型的贴边等待体态

### P1：再做“互动一下就感觉不同”

- 三种形象各自的拍准反馈
- 三种形象各自的拍偏反馈
- 三种形象各自的错过反馈

### P2：最后做“停在桌上也有存在感”

- 三种形象各自的待机小动作
- 三种形象各自的低侵扰动态氛围
- 更细的局部器官差异和挂件差异

## 验收标准

### 设计验收

- 用户不看文案，也能说出“这只像仓鼠 / 像小龙 / 像会贴人的精灵”
- 三种形象截一张静帧图，也不会被误认为只是同款换色

### 互动验收

- 同样是拍准，三种形象的回馈观感明显不同
- 同样是错过，三种形象的失落方式明显不同

### 产品验收

- 用户愿意开始用“那只仓仓的”“那只会冲的”“那只会贴的”来记住它们
- 用户会因为想看不同形象的反应，愿意多互动几次

## 情绪谱系

V1 建议先固定 8 个基础情绪态，每个情绪态对应一组动作、表情和声音。

### 1. 安静期待

用于：

- 它已经开始等你
- 但还没有明显撒娇

动作：

- 身体微微前倾
- 停顿更久
- 尾巴慢慢轻扫

表情：

- 眼睛更专注
- 耳朵轻轻向前

声音：

- 默认无声
- 偶发极轻的鼻音

### 2. 轻度亲近

用于：

- 它愿意靠近你
- 想被轻拍或顺手回应

动作：

- 小步靠近
- 轻贴边缘
- 身体轻轻蹭一下

表情：

- 眼神柔和
- 嘴角略松
- 尾巴更圆润

声音：

- 很轻的呼噜前奏

### 3. 满足释放

用于：

- 被拍对
- 情绪和互动能被正确释放

动作：

- 身体明显松开
- 翻肚皮、打转或贴近
- 出现余波动作，不立刻停

表情：

- 眼睛变圆或眯起来
- 耳朵立得更自然
- 尾巴摆动更流畅

声音：

- 极短满足音

### 4. 别扭未拍准

用于：

- 你拍了，但没拍在它想要的点上

动作：

- 身体先接住，再卡一下
- 尾巴顿住
- 毛没完全顺开

表情：

- 眼神还看着你，但没彻底放松
- 耳朵略偏

声音：

- 一声短短的顿音

### 5. 小失落

用于：

- 它等你，但没等到
- 亲和期待落空

动作：

- 慢慢缩回去
- 转身
- 蜷起来

表情：

- 视线下落
- 耳朵微垂
- 尾巴收一点

声音：

- 默认无声
- 偶发极轻的叹息感短音

### 6. 躁劲未散

用于：

- 它想释放躁动，但没被接住
- 或者刚处于高压仓、高躁动态

动作：

- 抖毛
- 重心不稳
- 小范围来回

表情：

- 眼神更锐
- 耳朵更紧
- 尾巴炸开或绷直

声音：

- 轻短的绷住感音色

### 7. 自我整理

用于：

- 它在清理自己
- 它在整理残渣、情绪或压仓感

动作：

- 低头舔毛
- 抖毛
- 原地顺身体

表情：

- 注意力向内收
- 与用户的目光接触减少

声音：

- 可选极轻细碎整理音

### 8. 高质量兴奋

用于：

- 吃到特制粮
- 触发惊喜节点
- 解锁新阶段

动作：

- 轻蹦
- 转圈
- 尾巴更亮地摆动

表情：

- 眼睛发亮
- 耳朵完全竖起

声音：

- 亮一点的庆祝音，但仍保持克制

## 成长偏向如何改变动作

相同情绪，在不同成长偏向下必须有不同身体语言。

### 细养型

关键词：

- 轻
- 慢
- 贴边
- 安静试探

动作特征：

- 靠近更慢
- 停顿更多
- 更常贴边等你
- 不会一下扑过来

表情特征：

- 眼神更柔
- 耳朵变化幅度小
- 尾巴轻扫而不是大摆

声音特征：

- 更轻、更短、更像试探

### 舒展型

关键词：

- 匀
- 顺
- 自然
- 稳定

动作特征：

- 节奏最圆润
- 靠近和离开都不突兀
- 互动余韵最自然

表情特征：

- 眼、耳、尾变化最平衡

声音特征：

- 最圆润、最自然

### 猛长型

关键词：

- 利落
- 发劲
- 外放
- 节奏更强

动作特征：

- 更常巡桌
- 站定更突然
- 尾巴和头部动作更有力
- 被拍对后释放更明显

表情特征：

- 眼神更亮
- 抬头动作更明显
- 耳朵更立

声音特征：

- 更亮、更直接

### 胀仓型

关键词：

- 闷
- 绷
- 先整理自己
- 带一点自我消化感

动作特征：

- 更常抖毛
- 靠近前会先整理
- 身体重心略沉
- 释放情绪时像“先卡一下再松”

表情特征：

- 嘴部更容易抿住
- 耳朵更容易后缩
- 尾巴更容易卷住或绷紧

声音特征：

- 更闷、更短
- 像把气从身体里轻轻放出来

## V1 动作库建议

V1 不要贪多，优先把最关键的 10 个动作做深。

### A. 基础待机 4 个

- 轻呼吸待机
- 贴边待机
- 巡桌待机
- 整理待机

### B. 关系关键动作 4 个

- 等你时
- 被拍对时
- 被拍偏时
- 没等到你时

### C. 成长关键动作 2 个

- 吃到高质量粮时
- 压仓后自我整理时

这 10 个动作如果成立，V1 的生命感就已经够强。

## 动作设计方法

每个动作都建议按同一模板设计，避免后面越做越散。

一个动作应至少定义：

1. 触发条件
2. 情绪名称
3. 主姿态
4. 节奏长度
5. 眼、耳、尾变化
6. 是否带声音
7. 不同成长偏向的差异

推荐格式示例：

- 动作名：等你轻拍
- 触发：有亲和期待，且暂未被回应
- 情绪：安静期待
- 主姿态：身体前倾，尾巴轻扫
- 节奏：1.2 秒循环
- 表情：眼神聚焦，耳朵向前
- 声音：无
- 偏向差异：
  - 细养型更贴边
  - 猛长型更抬头
  - 胀仓型会先抖一下毛再等

## 声音设计边界

V1 声音建议只做 5 个音色事件：

1. 轻满足
2. 强满足
3. 拍偏顿音
4. 轻求回应
5. 整理自己

V1 暂不做：

- 长音频
- 拟人语言
- 持续背景声
- 高频提醒声

## 实现顺序建议

这部分的正确落地顺序应固定为：

1. 先做动作节奏差异
2. 再做眼、耳、尾差异
3. 再做成长偏向分化
4. 最后补声音

原因是：

- 没有动作节奏，声音只会显得吵
- 没有成长偏向差异，动作再多也会像模板动画
- 没有表情细节，关系感会停留在文案层

## 当前阶段的具体建议

当前最值得继续做的，不是再扩系统，而是把下面 3 件事做稳：

1. 等你、拍准、拍偏、没等到，这 4 个关系关键动作继续拉开差异。
2. 细养型、猛长型、胀仓型，至少各补 1 个独特待机动作和 1 个独特求拍动作。
3. 先做一套极短、可关闭的声音试验版，只在关键关系节点触发。

如果这三层做出来，阿在就会从“有设计理念的桌宠”真正走到“有感染力的电子宠物”。
