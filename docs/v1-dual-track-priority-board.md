# 阿在 Token Pet 双线并进优先级看板

## 结论

从现在开始，阿在 Token Pet 的版本推进按两条线并进：

1. 生命代谢线
2. 情绪价值线

这份文档不再回答“应该优化什么”，而直接回答：

- 先做什么
- 哪些是必须先做的 `P0`
- 哪些是下一层推进的 `P1`
- 哪些是暂时不抢优先级的 `P2`

## 看板使用规则

### `P0`

不做会直接影响产品成立，或直接打断用户对产品的信任与陪伴感。

### `P1`

做了会显著提升体验和留存，但不至于让当前版本完全失效。

### `P2`

重要，但当前不应抢走 `P0 / P1` 的资源。

## 一、生命代谢线看板

### `P0`

- Token 统计稳定性
  要点：来源稳定、时间归属、去重、漏记控制。
- Token 分类可信度
  要点：任务类型映射、质量信号映射、低置信度兜底。
- 转化可解释性
  要点：为什么转成这种粮、为什么是这个量级，用户要看得懂。
- 代谢与消耗稳定性
  要点：自动吃粮、库存扣减、互动能量积累、极端状态边界不能乱。
- 平衡底线
  要点：不能出现“多用 Token 永远最好”或“某一种回应永远最优”。

### `P1`

- 转化精度提升
  要点：不同工作风格对粮食结构和能量结构的差异更明显。
- 代谢节奏自然化
  要点：吃、积攒、释放、恢复更像生命过程，不像脚本扣数。
- 生长差异显性化
  要点：成长变化不只写在状态里，要能被用户看见。
- 关系历史进入养成
  要点：关系不只影响情绪，还真正影响成长方向。

### `P2`

- 更复杂的外部来源接入
- 更细粒度分类器
- 更复杂的营养/毒性子系统
- 更长链路的高级成长分支

## 二、情绪价值线看板

### `P0`

- 第一印象成立
  要点：首次诞生要让人记住这只不是默认模板。
- 期待感成立
  要点：用户要能看出它现在是不是在等、想要什么样的回应。
- 回应感成立
  要点：拍准、拍偏、错过必须形成明显心理差异。
- 陪伴感不被破坏
  要点：不能闪、不能乱抢前台、不能高频打扰。

### `P1`

- 被记住感增强
  要点：它开始表现出记得用户过去怎么对它。
- 个体关系感增强
  要点：用户开始能说出“我这只的脾气是什么样”。
- 状态文案人格化
  要点：短句更像它自己在表达，而不是系统提示。
- 上手体验温柔化
  要点：新用户能更自然进入关系，而不是只完成安装。

### `P2`

- 更多声音层设计
- 更复杂的情绪细分谱系
- 更丰富的长期关系仪式感节点
- 更展开的图鉴叙事系统

## 三、当前版本对应优先级

## `v1.0.x`

### 生命代谢线

- `P0`：统计稳定、转化稳定、代谢稳定、基础平衡底线
- `P1`：可解释性增强

### 情绪价值线

- `P0`：陪伴感不被稳定性问题打断
- `P1`：基础期待感和回应感清晰可见

## `v1.1.x`

### 生命代谢线

- `P0`：不破坏底层解释性
- `P1`：转化结果和状态结果更容易被用户理解

### 情绪价值线

- `P0`：第一印象、期待感、回应感
- `P1`：状态文案人格化、首次上手温柔化

## `v1.2.x`

### 生命代谢线

- `P0`：关系历史真正进入成长与养成
- `P1`：成长差异显性化

### 情绪价值线

- `P0`：被记住感、个体关系感
- `P1`：更稳定的长期陪伴节奏

## `v1.5.x`

### 生命代谢线

- `P0`：外部用户也能理解这条链怎么工作

### 情绪价值线

- `P0`：陌生用户首次使用就能感到陪伴价值

## 四、当前最值得马上推进的事项

### 生命代谢线

- `P0`：长时统计与转化观察
- `P0`：代谢边界与库存消耗回归
- `P0`：平衡底线检查，避免单一最优解
- `P1`：转化结果与状态变化的解释性展示

### 情绪价值线

- `P0`：首次诞生体验
- `P0`：认主完成反馈
- `P0`：进食、等待、拍准、拍偏、错过差异
- `P1`：状态文案与轻提示
- `P1`：首次上手说明

## 五、当前不建议抢优先级的事项

- 多宿主铺开
- 复杂商业化设计
- 完整图鉴宇宙
- 大量新动作面板
- 复杂排行或社交传播层
- 过度细碎但解释不清的规则系统

## 当前判断

后面如果要判断一个想法要不要做，最简单的方式不是问“它酷不酷”，而是问：

1. 它是在推进生命代谢线的 `P0 / P1` 吗？
2. 它是在推进情绪价值线的 `P0 / P1` 吗？

如果两个问题都答不上来，它当前就不该抢优先级。
