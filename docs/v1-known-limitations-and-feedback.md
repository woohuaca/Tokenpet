# 阿在 Token Pet `v1.0.0` 已知限制与反馈重点

## 为什么要单独列这份文档

正式首发版最怕两件事：

- 用户把“当前首发范围”误读成“已经全平台完成”
- 团队拿到很多零散感受，但收不到真正有决策价值的反馈

所以这份文档只做两件事：

- 明确当前已知限制
- 明确当前最希望用户反馈什么

## 当前已知限制

### 1. 平台范围有限

- 当前主要面向 `Codex + macOS`
- 其他宿主和其他系统暂不作为当前正式发布范围

### 2. 正式范围仍有限

- 当前已经按正式版口径发布到 Codex
- 但当前不等于已经具备公开插件商店、多宿主、多系统的完整正式分发能力

### 3. 任务分类仍有启发式成分

- 当前任务类型推断已可用
- 但还不是最终高精度分类器

### 4. 长时稳定性还在继续观察

- 启动链、单实例和短周期重复启停已经通过当前验证
- 但更长时段的空闲、自启、跨桌面空间和异常恢复仍在继续观察

### 5. 重置不会自动备份

- `reset_pet_data.command` 会直接清掉当前这只的成长与关系
- 当前没有自动备份恢复能力

### 6. 视觉与动作还在 V1 收口中

- 当前已经有明显的状态、成长偏向和关系差异
- 但动作强弱、表情层和首次迎接感仍有继续打磨空间

## 当前不希望用户误解的点

### 1. 这不是效率工具面板

它的核心不是更好地显示数字，而是把 Token 消耗翻译成一种生命反馈。

### 2. 这不是“多用 Token 就更好”

当前设计强调的是：

- 多用会长得更快，但也可能更躁、更重
- 少用会长得更慢，但也可能更细、更敏感

我们想观察的不是“用户会不会多花 Token”，而是“这种成长价值观是否成立”。

### 3. 这不是已经定型的最终宠物形象

当前版本已经具备“不是默认模板”的差异，但还不是最终完全展开的形态系统。

## 当前最希望收集的反馈

### 一、生命感

最想知道：

- 它有没有让你觉得像个小生命，而不是一个统计挂件？

### 二、独特性

最想知道：

- 你会不会感觉“这只是我养出来的”？

### 三、资源感知

最想知道：

- 它有没有让你更直觉地意识到 Token 背后是资源和能源消耗？

### 四、关系驱动

最想知道：

- 你会不会因为它表达期待、被拍准或拍偏，而开始在意它？

### 五、打扰度

最想知道：

- 它是在陪伴你，还是在打扰你？

## 建议当前用户反馈时至少回答的 5 个问题

1. 第一次看到它时，你觉得它像不像一只“被生成出来的宠物”？
2. 你能不能自然理解 `Token -> 粮食 -> 生长` 这条逻辑？
3. 你会不会在工作间隙想看它一眼？
4. 你能不能感到拍准、拍偏、错过之间的差异？
5. 如果让你继续养一周，你最期待它长出什么？

## 当前判断

这次 `v1.0.0` 已经可以作为 `Codex + macOS` 范围内的正式首发版使用，但还不适合被误读为“多平台最终完成版”。

因此，收反馈时最重要的不是问：

- “它是不是已经把所有渠道都做完了？”

而是问：

- “这条产品方向在正式首发范围内是否已经成立？”
