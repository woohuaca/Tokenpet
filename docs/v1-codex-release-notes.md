# 阿在 Token Pet `v1.0.0` Codex 首发正式版发布说明

## 这次发布的结论

这次不是再发一份 `dev preview`，而是把阿在 Token Pet 收口为：

- `Codex-first`
- `macOS companion`
- `本地可安装、可发现、可启动` 的正式首发版

这里的“正式发布”有清晰边界：

- 它已经按正式版口径进入 Codex 发布链
- 但当前正式范围仍限定在 `Codex + macOS`
- 这不等于已经进入公开插件商店

## 这次正式发布解决了什么

相比此前的 `dev preview`，这次发布主要解决三件事：

1. 发布定位收口。
   从“开发者预览”改为“Codex 首发正式版”，避免用户把当前链路误读为仍只适合内部验证。
2. 发布工件收口。
   新增正式版构建脚本，生成稳定命名的 `v1.0.0` 发布包，而不是继续沿用 `dev preview` 产物。
3. 发布链稳定性收口。
   启动与打包脚本补了更稳的 SDK 解析逻辑，减少本机 Swift/SDK 版本别名导致的编译失败。

## 当前正式发布范围

- 宿主：Codex
- 系统：macOS
- 形态：插件入口 + 桌面 companion
- 启动入口：`start.command`
- 插件首条 starter prompt：`启动阿在 Token Pet 桌宠`

## 当前正式发布已经成立的能力

- 真实 Token 会先转成粮食，再驱动宠物状态变化
- 宠物会自动吃粮，并逐步积累互动能源
- 首次诞生、认主、拍准、拍偏、错过已有明确反馈差异
- 单实例启动链与 bridge 自愈链已经收口
- 发布到 Codex 的脚本、缓存路径与正式发布包命名已经统一

## 当前不包含什么

这次正式发布不代表：

- 已经完成多宿主正式接入
- 已经具备公开插件商店分发能力
- 已经做完安装器、备份恢复和完整图鉴宇宙

这些仍属于下一阶段。

## 对外该怎么描述

最准确的一句话是：

- 阿在 Token Pet 已经以 `Codex + macOS companion` 为首发范围进入正式发布。
