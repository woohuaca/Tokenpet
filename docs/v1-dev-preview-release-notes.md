# 阿在 Token Pet `v0.1.0-dev-preview` 发布说明

## 这是什么

这是阿在 Token Pet 的第一份 `dev preview`。

它不是正式稳定版，而是一份用来验证核心体验是否成立的开发者预览：

- 真实 Token 是否真的能养活一只桌边宠物
- 它是否真的像一个会回应、会记住你、会慢慢长成“你的那只”的生命体
- 这种形态是否值得继续进入更公开的下一阶段

## 当前版本已经成立的体验

### 1. 真实 Token 转粮

- 宠物吃的不是虚构分数，而是你在 Codex 工作中真实花掉的 Token
- Token 会先转成不同类型的粮食，再由宠物慢慢代谢

### 2. 一只会长出来的宠物

- 首次启动会生成名字、灵魂描述、外观倾向和出生叙事
- 它不是固定模板，而会被饮食结构、关系和沟通风格慢慢塑形

### 3. 单一互动“拍一下”

- 当前版本只保留一个动作：拍一下
- 拍准、拍偏、错过、认主，都会带来不同反馈

### 4. 桌面 companion 体验

- 以桌面悬浮宠物作为主体体验
- 当前版本已经通过单实例与短周期重复启停回归

## 当前版本最适合谁

更适合下面这类用户：

- 经常在 Codex 里工作的人
- 愿意安装本地工具的人
- 能接受 `developer preview` 稳定度的人
- 对“AI 工作痕迹如何长成宠物”这件事本身有兴趣的人

## 当前版本不是什么

请不要把它理解成：

- 正式稳定版插件
- 面向所有平台的公开版本
- 已完成的完整桌宠宇宙
- 一套追求高完成度美术包装的最终产品

它当前更准确的定位是：

- 核心生命体验已经成立
- 关键桌面稳定性已经基本收住
- 适合进入小范围私测

## 本轮最值得重点体验的部分

建议私测时重点感受这几件事：

1. 第一次看到它时，是否觉得“这只不是默认宠物”。
2. 它吃到真实 Token 后，是否真的像在自然生长，而不是在涨一个数字。
3. 它表达期待、等待你、被你拍准或拍偏时，是否有关系感。
4. 连续用一段时间后，是否会开始在意“它接下来会长成什么样”。

## 当前版本的关键变化

相比更早的原型版，这份 `dev preview` 已经补上了：

- 首次诞生初始化
- 单一交互“拍一下”
- 成长偏向与关系沉淀
- 极简主界面与图鉴/状态分层
- 单实例启动链
- bridge 改由桌宠本体自愈拉起

## 私测时最希望得到的反馈

如果只收 4 类反馈，最有价值的是：

1. 它有没有让你觉得“活着”？
2. 它有没有让你觉得“这是我的这只”？
3. 它有没有让你更感知到 Token / 算力 / 资源消耗？
4. 你会不会在工作间隙下意识回头看它一眼？

## 下一步会怎么走

如果这轮私测结果成立，后续会优先推进：

1. 更完整的首次迎接感
2. 更强的动作强弱与表情差异
3. 更贴近日常使用的长时稳定性
4. 更明确的发布安装体验
