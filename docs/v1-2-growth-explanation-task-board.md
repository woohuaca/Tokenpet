# 吞金兽 Token Pet `v1.2.x` 生长养成解释面板任务清单

## 结论

`v1.2.x` 的开发不应从“再做什么新效果”开始，而应从“把解释层做成一个完整闭环”开始。

这一版建议按三类任务推进：

1. 解释骨架
2. 用户可读表达
3. 一致性与回归

## 一、`P0`

### 任务 1：今日吞食摘要成形

目标：

- 让用户一眼看懂今天主要吃了什么

预期产出：

- 今日 Token 总量摘要
- 今日主任务类型
- 今日主质量结构
- 今日主粮食来源

验收点：

- 用户能快速说出“它今天主要吃的是哪类工作”

### 任务 2：代谢结果解释

目标：

- 让用户知道这些 Token 主要转成了什么状态结果

预期产出：

- 一句代谢结果结论
- 一句当前风险或偏向说明

验收点：

- 用户能理解“它为什么现在偏猛 / 偏胀 / 偏累 / 偏想靠近”

### 任务 3：当前长势判断展示

目标：

- 把系统内的长势倾向翻译成用户能懂的话

预期产出：

- 长势名称
- 长势原因
- 长势影响

验收点：

- 用户会开始把这只吞金兽看成近期工作方式养出来的结果

### 任务 4：当前情绪原因展示

目标：

- 让等待、靠近、别扭、收回去这些情绪更像有原因的状态

预期产出：

- 当前情绪判断
- 当前请求
- 情绪成因解释

验收点：

- 用户知道它为什么现在想被这样回应

### 任务 5：下一步建议

目标：

- 让解释闭环真正落到行动

预期产出：

- 一条当前最推荐动作
- 一条简短依据

验收点：

- 用户看完后知道下一步做什么

## 二、`P1`

### 任务 6：解释文案人格化

目标：

- 让解释像吞金兽自己在表达，不像系统播报

预期产出：

- 一版解释句模板
- 一版不同长势 / 情绪下的语气差异

### 任务 7：解释与图鉴分层

目标：

- 让“长期信息”和“当前解释”在信息层级上分开

预期产出：

- 图鉴里的长期档案区
- 当前解释区

### 任务 8：阶段性变化提示

目标：

- 让用户在长势明显变化时收到一次轻提示

预期产出：

- 长势切换提示
- 仓压 / 舒展 / 细养 / 猛长变化提示

## 三、`P2`

### 任务 9：阶段性自我叙事

目标：

- 在规则解释稳定后，再让它开始生成更像个体故事的表达

### 任务 10：周度成长报告

目标：

- 用周尺度回答“这一周我把它养成了什么样”

## 建议开发顺序

### 第一批

- 任务 1：今日吞食摘要成形
- 任务 2：代谢结果解释
- 任务 3：当前长势判断展示
- 任务 4：当前情绪原因展示
- 任务 5：下一步建议

原因：

- 这五项合在一起，才是一个完整解释闭环

### 第二批

- 任务 6：解释文案人格化
- 任务 7：解释与图鉴分层
- 任务 8：阶段性变化提示

原因：

- 先让解释成立，再让解释更像它自己、更好读

### 第三批

- 任务 9：阶段性自我叙事
- 任务 10：周度成长报告

原因：

- 这两项建立在解释骨架稳定之后，否则只会增加漂亮但不稳的输出

## 最小交付面

如果 `v1.2.x` 只能先做最小交付面，建议至少完成：

- 任务 1
- 任务 2
- 任务 3
- 任务 4
- 任务 5

这五项是 `v1.2.x` 的最小可成立版本。

## 当前判断

`v1.2.x` 真正的任务重点，不是做出更复杂的系统，而是：

- 把已有系统解释出来
- 把解释变成用户理解与回应的入口

只要这一层做准，吞金兽后面的长期养成和个体关系感，就会更像在自然长出来。
