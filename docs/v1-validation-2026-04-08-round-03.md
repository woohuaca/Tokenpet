# 阿在 Token Pet V1 第三轮验证记录

## 本轮目的

这轮验证不再扩玩法，而是专门补 `V1` 发布前最需要的一类证据：

- 短周期重复启停回归
- 停止后是否真的清空
- 重新启动后是否稳定回到 `1` 只桌宠 + `1` 条 bridge
- `desktop-pet.pid` 与 `codex-bridge.pid` 是否持续与真实进程一致

目标是回答：

- 当前版本是否已经通过“多次日常启停不回退”的短周期验证
- 是否可以把启动 / 单实例从“专项通过”提升为“可进入私测前回归观察”

## 本轮结论

- 总结论：`通过`
- 是否建议进入小范围私测准备：`是`
- 本轮关键结论：
  - 连续 3 轮 `停止 -> 启动 -> 等待` 后，均稳定恢复为 `1` 只桌宠 + `1` 条 bridge
  - 每轮 `desktop-pet.pid` 与 `codex-bridge.pid` 都与真实进程一致
  - 当前启动链已经具备短周期回归稳定性

这意味着：

- `V1` 当前已经不再被“启动 / 单实例”阻塞
- 发布前仍建议保留一轮更贴近日常使用的长时观察，但它已不再是私测前 blocker

## 本轮验证方法

验证环境：

- 设备：当前开发机
- 系统：macOS
- 宿主：Codex + 本地桌面 companion
- 验证日期：2026-04-08

验证动作：

1. 执行 `scripts/stop_desktop_pet.command`
2. 等待 1 秒后检查真实进程列表
3. 执行 `scripts/launch_desktop_pet.command`
4. 等待 2 秒后检查真实进程列表
5. 读取 `runtime/desktop-pet.pid` 与 `runtime/codex-bridge.pid`
6. 连续重复 3 轮
