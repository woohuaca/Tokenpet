# 阿在 Token Pet 核心环节持续优化方案

## 结论

阿在 Token Pet 的需求，至少要沿两条线并进：

1. 持续优化 `Token -> 分类 -> 转化 -> 代谢 -> 平衡 -> 生长养成` 这条生命代谢线。
2. 持续优化宠物的情绪价值线。

这份文档定义的是第一条线。

阿在 Token Pet 后续所有版本，不应只优化“看起来更可爱”或“启动更稳”，而要持续优化同一条底层主循环：

- Token 统计
- Token 分类
- 能量转化
- 代谢与消耗
- 平衡控制
- 生长与养成
- 情绪互动

这条链路是产品真正成立的骨架。

如果其中任何一环长期失真，产品就会退化成：

- 统计挂件
- 随机宠物
- 靠文案和动画硬撑的桌面玩具

所以，版本优化必须始终围绕这条链，并与情绪价值线并进，而不是围绕零散功能点。

## 为什么要单独把这条链拎出来

当前我们已经把版本路线定出来了，但如果没有一条持续优化主线，后面每个版本很容易各做各的：

- 稳定性只修启动
- 体验版只修动作
- 养成版只加状态

结果就是每一版都看起来在前进，但整体生命感不一定在变强。

因此，需要把真正要持续优化的核心问题固定为：

1. Token 是否被正确感知。
2. Token 是否被正确理解。
3. Token 是否被正确转成宠物世界里的能量。
4. 能量是否被正确代谢，不会只变成数字堆叠。
5. 平衡是否健康，不会导向单一最优玩法。
6. 生长与养成是否真的能沉淀成个体差异。
7. 情绪互动是否真的把前面所有变化表达出来。

与之并列的第二条线见：`docs/v1-emotional-value-optimization-plan.md`

## 核心链路定义

建议以后统一用下面这条链表达产品主循环：

`Token 统计 -> Token 分类 -> 转化为粮食与能量 -> 代谢与消耗 -> 状态平衡 -> 生长与养成 -> 情绪互动 -> 反过来影响用户下一轮回应`

这比简单说 `Token -> 粮食 -> 吃 -> 拍 -> 成长` 更完整，因为它把中间真正决定质量的环节都展开了。

## 七个环节的持续优化目标

## 一、Token 统计

### 这一环在解决什么

先确保宠物吃到的，真的是用户真实工作产生的 Token，而不是虚构积分。

### 长期优化目标

- 统计来源更稳定
- 时间归属更准确
- 重复事件更少
- 漏记事件更少

### 这一环如果做不好，会发生什么

- 用户不相信它吃的是真实 Token
- 产品失去“资源感知”这个独特价值
- 后面所有转化、成长和情绪都像假演出

### 后续持续优化方向

- 来源接入稳定性
- 事件去重
- 时间窗口对齐
- 统计异常回溯能力

## 二、Token 分类

### 这一环在解决什么

不是所有 Token 都该变成同一种粮食。分类决定宠物吃进去的“是什么”，而不仅是“多少”。

### 长期优化目标

- 分类结果更可解释
- 分类准确率逐步提高
- 低置信度分类有更稳的兜底
- 分类粒度不过细，不过拟合

### 这一环如果做不好，会发生什么

- 用户会觉得粮食类型是随机的
- 宠物成长失去工作风格映射
- 养成差异会越来越假

### 后续持续优化方向

- 任务类型映射规则
- 质量信号映射
- 低置信度 fallback 规则
- 分类透明度展示

## 三、能量转化

### 这一环在解决什么

Token 不能直接变经验值，必须先被翻译成宠物世界可理解的粮食、营养和互动能源。

### 长期优化目标

- 转化规则更稳定
- 高质量输入和普通输入的差异更合理
- 不同类型工作对能量结构影响更明显
- 用户能看懂“为什么这次转出来的是这种粮”

### 这一环如果做不好，会发生什么

- Token 又退化成抽象数字
- 粮食系统变成换皮经验条
- 产品失去“珍惜资源”的感受

### 后续持续优化方向

- 转粮比例
- 粮食类型与质量映射
- 互动能量积累逻辑
- 转化结果可见性

## 四、代谢与消耗

### 这一环在解决什么

这里对应你说的“消灭”环节，我这里按“宠物对库存、营养和互动能量的消化与消耗”来定义。

宠物不能只是被动囤粮，它必须自己吃、自己代谢、自己释放，不然生命感立不住。

### 长期优化目标

- 自动吃粮更自然
- 代谢节奏更像生命过程，而不是脚本扣数
- 能量释放更符合情绪状态
- 不出现库存异常、过快耗尽或长期卡死

### 这一环如果做不好，会发生什么

- 宠物像仓库，不像生命体
- 用户看不出“它真的在活”
- 状态变化会变得生硬

### 后续持续优化方向

- 自动吃粮触发阈值
- 库存消耗节奏
- 情绪能量释放规则
- 高压、高仓、低能等极端状态边界

## 五、平衡控制

### 这一环在解决什么

产品不能导向“多用 Token 就总是更好”，也不能导向“只要摸对一次就永远最优”。平衡层要确保成长价值观成立。

### 长期优化目标

- 高产、高质、克制、细腻几种风格都能成立
- 没有单一最优养法
- 高强度使用既有收益也有代价
- 低强度使用既有克制感也不会彻底养不活

### 这一环如果做不好，会发生什么

- 产品会被攻略化
- 用户只会追求数值最大化
- 生命感会输给最优解

### 后续持续优化方向

- 饱食、毒性、情绪、关系之间的制衡
- 高质量与高数量的权重关系
- 互动回报强度曲线
- 不同成长风格的平衡校正

## 六、生长与养成

### 这一环在解决什么

前面所有输入，最后都必须沉淀成“这只是我养出来的”。

### 长期优化目标

- 成长不是固定进化树
- 关系历史会持续塑造外观、口吻和习惯
- 不同用户长期养出的个体差异更明显
- 生长变化能被用户感知，而不是只存在于数据层

### 这一环如果做不好，会发生什么

- 产品短期新鲜，长期乏力
- 用户不会建立持续陪伴理由
- “独特性”会停留在名字和几句文案

### 后续持续优化方向

- 关系历史参与度
- 外观差异显性化
- 阶段性自我解释
- 称号、习惯、期待偏好生成

## 七、情绪互动

### 这一环在解决什么

情绪互动不是单独的一层皮，而是把前面六环的变化翻译成用户能感受到的东西。

### 长期优化目标

- 进食、等待、拍准、拍偏、错过差异更清晰
- 文案、动作、轻特效共同表达情绪
- 情绪反馈能反映真实状态，不是假装有情绪
- 打扰度始终可控

### 这一环如果做不好，会发生什么

- 前面的成长与平衡即使存在，用户也感受不到
- 桌宠会退化成“后台算法 + 前台待机动画”

### 后续持续优化方向

- 关键情绪时刻的差异强化
- 轻提示频率控制
- 动作与状态的一致性
- 关系阶段变化的可见反馈

## 这条线如何映射到当前版本路线

### `v1.0.x`

优先守住：

- Token 统计
- 转化稳定性
- 代谢与消耗稳定性
- 基础情绪反馈可见性

### `v1.1.x`

优先做深：

- 进食、等待、拍准、拍偏、错过的情绪表达
- 状态文案与轻提示
- 首次诞生和认主这两个关键关系节点

### `v1.2.x`

优先做深：

- 生长与养成
- 情绪互动与关系历史的联动
- 个体差异的持续沉淀

### `v1.5.x`

优先守住：

- 统计、分类、转化规则的外部可解释性
- 安装、恢复、反馈与问题排查链

### `v2.0.0`

优先扩展：

- 更多来源
- 更多宿主
- 更正式的分发与更新链

但不能破坏前面的主循环质量。

## 后续每版都应回答的检查问题

每次准备做版本规划或需求裁剪时，都建议先回答下面七个问题：

1. 这版是否让 Token 统计更真实？
2. 这版是否让 Token 分类更可信？
3. 这版是否让能量转化更可感知？
4. 这版是否让代谢与消耗更像生命过程？
5. 这版是否让平衡更健康，而不是更可攻略？
6. 这版是否让生长与养成更像“我自己的这只”？
7. 这版是否让情绪互动更能把前面六项表达出来？

如果一版和这七个问题都没有明显关系，那它就不应成为当前优先版本的核心任务。

## 当前判断

阿在 Token Pet 未来要不断优化的，不是某一个局部功能，而是整条生命化代谢链。

只有把：

- 真实 Token
- 可解释分类
- 合理转化
- 自然代谢
- 健康平衡
- 独特成长
- 情绪互动

持续串成一条越来越可信的链，阿在 Token Pet 才会越来越像一个真正可长期养成的产品。

但它要真正被人持续喜欢，还必须和情绪价值线一起推进。
