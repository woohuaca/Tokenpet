# 阿在 Token Pet 情绪价值持续优化方案

## 结论

阿在 Token Pet 的需求，至少要沿两条线并进：

1. 持续优化 `Token -> 分类 -> 转化 -> 代谢 -> 平衡 -> 生长养成` 这条生命代谢线。
2. 持续优化宠物的情绪价值线。

这份文档定义的是第二条线。

如果第一条线决定“它是不是真的活着”，那第二条线决定“用户会不会想继续陪着它”。

没有情绪价值，宠物会退化成：

- 一个会动的资源面板
- 一个能养但不让人牵挂的桌面对象
- 一个逻辑成立但不让人心软的系统

所以，情绪价值不是装饰层，而是陪伴产品的核心价值层。

## 为什么要把情绪价值单独立线

如果只强调主循环稳定和规则可信，产品容易走向：

- 统计更准
- 分类更稳
- 状态更多

但用户仍然可能不会更喜欢它。

因为用户是否会继续养，不只取决于它有没有系统逻辑，还取决于：

1. 它会不会让我心里一软。
2. 它会不会让我觉得被等着、被记得、被回应。
3. 它会不会在工作缝隙里给我一点值得回头看的情绪价值。

因此，情绪价值不能被当成“做完主循环以后再顺手修一修”的附属项，而应该作为并进主线持续优化。

## 情绪价值主线定义

建议以后统一把这条线表达为：

`第一印象 -> 期待感 -> 回应感 -> 被记住感 -> 陪伴感 -> 个体关系感`

它和生命代谢线并列，不互相替代。

## 六个环节的持续优化目标

## 一、第一印象

### 这一环在解决什么

用户第一次看到它时，要不要在心里把它当成“一个存在”，而不是“一个组件”。

### 长期优化目标

- 初见时更有出生感
- 不是默认模板感更强
- 名字、气质、第一印象更容易被记住

### 如果做不好，会发生什么

- 用户的第一反应还是“装了一个桌宠”
- 后面再做再多关系反馈，也会少一层起点

## 二、期待感

### 这一环在解决什么

宠物不能只是等状态变化，它要让用户感到“它现在有点想要我怎么回应它”。

### 长期优化目标

- 等待更有方向
- 期待更可感知
- 不同个体的等待方式更不一样

### 如果做不好，会发生什么

- 用户看不出它什么时候真的在等
- “拍一下”会退化成随机点击

## 三、回应感

### 这一环在解决什么

用户的动作要能被宠物明确接住，拍准、拍偏、错过必须形成心理差异。

### 长期优化目标

- 拍准更有被接住感
- 拍偏更有别扭感
- 错过更有落空感
- 认主是明显关系节点，不是普通交互

### 如果做不好，会发生什么

- 用户不会在意自己刚刚有没有回应对
- 关系不会成立，只会剩操作反馈

## 四、被记住感

### 这一环在解决什么

用户要慢慢感觉到：这只不是每次都重新认识我，它记得我过去怎么对它。

### 长期优化目标

- 文案和行为都带有关系历史痕迹
- 它对同一个人的回应越来越像“认得这个人”
- 称号、习惯、期待偏好逐渐稳定下来

### 如果做不好，会发生什么

- 长期养成不会沉淀出关系价值
- 用户会把它当成短期新鲜感产品

## 五、陪伴感

### 这一环在解决什么

宠物要成为工作缝隙里的柔性存在，而不是新的负担。

### 长期优化目标

- 陪伴感大于打扰感
- 存在感稳定，但不过度抢前台
- 文案、动作、提示频率不过载

### 如果做不好，会发生什么

- 用户会关掉它
- 它会从陪伴者变成干扰源

## 六、个体关系感

### 这一环在解决什么

最终用户要觉得：不是“我有一个 Token Pet”，而是“我在养我这只”。

### 长期优化目标

- 不同人养出来的关系气质不一样
- 个体性不只体现在名字，而体现在脾气、等待方式、反馈方式
- 用户能说出自己这只的性格特征

### 如果做不好，会发生什么

- 产品会停留在普遍可爱，而不是个体可爱
- 留存理由会明显变弱

## 这条线如何映射到当前版本路线

### `v1.0.x`

优先守住：

- 陪伴感不被启动、闪烁、丢失问题打断
- 基础期待感和基础回应感已经可见

### `v1.1.x`

优先做深：

- 第一印象
- 期待感
- 回应感

也就是你当前定下来的五个模块。

### `v1.2.x`

优先做深：

- 被记住感
- 个体关系感

### `v1.5.x`

优先守住：

- 陌生用户也能在首次使用时感受到陪伴，而不是只完成安装

### `v2.0.0`

优先扩展：

- 在更多宿主和更多系统上复制情绪价值，而不是只复制功能链

## 后续每版都应回答的检查问题

1. 这版有没有让第一印象更成立？
2. 这版有没有让宠物的期待更可感知？
3. 这版有没有让回应差异更有情绪含义？
4. 这版有没有让用户感到自己被记住？
5. 这版有没有让陪伴感提升，而不是打扰感提升？
6. 这版有没有让“我的这只”更明显？

如果一版和这六个问题都没有明确关系，那它就不应被当作情绪价值主线上的核心版本工作。

## 当前判断

阿在 Token Pet 后续至少要并进两条线：

- 一条让它越来越像一个真实被 Token 驱动的生命体
- 一条让它越来越像一个值得继续陪伴的情绪对象

只有这两条线一起推进，产品才不会变成：

- 只有系统，没有感情
- 只有可爱，没有骨架
