# 阿在 Token Pet `v1.1.x` 开发任务清单

## 结论

`v1.1.x` 的开发不应再按零散模块推进，而应按双线任务板推进：

1. 生命代谢线
2. 情绪价值线

这一版的总体原则是：

- 情绪价值线主攻
- 生命代谢线兜底

也就是说，`v1.1.x` 主要负责让用户第一次更容易被打动，但不能以破坏底层 `Token -> 能量 -> 养成` 的解释性为代价。

## 使用方式

### 任务状态建议

- `未开始`
- `进行中`
- `已完成`
- `延后`

### 优先级定义

- `P0`：不做会直接影响 `v1.1.x` 成立
- `P1`：建议纳入本版，但可在 `P0` 之后推进
- `P2`：记录下来，不抢当前版本资源

## 一、生命代谢线任务

## `P0`

### 任务 1：转化结果解释性收口

目标：

- 让用户更容易理解“为什么这次是这种粮、这种量级”

预期产出：

- 转粮摘要文案优化
- 粮食类型、来源、质量之间关系更清楚
- 状态面板里关键转化信息更易读

验收点：

- 用户能更自然复述“这次 Token 被转成了什么”
- 不会觉得粮食类型像随机抽出来

依赖：

- 当前分类和转化规则保持稳定

### 任务 2：代谢边界回归

目标：

- 确保自动吃粮、库存扣减、互动能量积累在 `v1.1.x` 里不会因为体验层改动出问题

预期产出：

- 一轮代谢边界验证
- 高库存、低库存、连续进食、连续等待几种场景回归

验收点：

- 不出现异常扣库存
- 不出现互动能量异常飙升或卡死
- 情绪反馈和底层状态基本一致

依赖：

- 当前自动吃粮和库存逻辑

### 任务 3：平衡底线检查

目标：

- 防止 `v1.1.x` 为了强化反馈，把某一类行为意外做成单一最优解

预期产出：

- 一份平衡检查结论
- 认主、拍准、拍偏、错过相关回报强度复核

验收点：

- 不会形成“某一种拍法永远最好”
- 不会形成“高 Token 永远直接更优”

依赖：

- 当前拍击与能量释放规则

## `P1`

### 任务 4：状态结果解释性展示

目标：

- 让用户更容易把“吃了什么、现在怎么了、为什么想被这样回应”连起来理解

预期产出：

- 状态文案、摘要文案、轻提示之间的分工更清楚

验收点：

- 用户更容易理解当前状态由什么引起

## 二、情绪价值线任务

## `P0`

### 任务 5：首次诞生体验重做

目标：

- 把“初始化完成”推进为“它刚刚出生”

预期产出：

- 一轮新的首次诞生演出
- 名字、出生描述、第一印象的出场顺序
- 一句更合适的初次引导话术

验收点：

- 新用户第一次看到时，能记住它是谁
- 新用户会觉得“这只不是默认模板”

依赖：

- 当前初始化字段与身份生成结果

### 任务 6：认主完成反馈强化

目标：

- 让第一次拍对并完成认主，成为真正的关系节点

预期产出：

- 一组认主专属动作或姿态
- 一组认主专属反馈文案
- 一段认主完成后的短余波反馈

验收点：

- 用户能明显区分认主成功和普通拍准
- 用户能理解“关系是建立出来的”

依赖：

- 当前认主判定逻辑

### 任务 7：进食 / 等待 / 拍准 / 拍偏 / 错过差异强化

目标：

- 拉开五类关键情绪时刻的动作和特效差异

预期产出：

- 五类状态的动作层级定义
- 对应轻特效或轻反馈差异

验收点：

- 不看说明时也能大致分清五类状态
- 不会因为特效增强而显著提高打扰度

依赖：

- 当前动作与状态映射

### 任务 8：陪伴感保护

目标：

- 强化反馈时，不让桌宠变得更吵、更烦、更抢前台

预期产出：

- 一轮打扰度检查
- 提示频率、动作幅度、特效强度的收敛结论

验收点：

- 增强了情绪表达后，仍然不会明显变成干扰源

依赖：

- 当前显示层与提示层稳定性

## `P1`

### 任务 9：状态文案人格化

目标：

- 让状态表达更像它自己在说，而不是系统在播

预期产出：

- 一版新的状态短句库
- 一版新的等待、进食、拍偏、错过后短提示

验收点：

- 文案更像生命提示
- 语气更稳定，不像人格跳变

### 任务 10：首次上手说明温柔化

目标：

- 让用户更自然进入关系，而不是只完成安装

预期产出：

- 更新 `README`
- 更新上手卡
- 更清楚的第一轮建议动作

验收点：

- 新用户能在 1 分钟内理解怎么开始养它

## 三、建议开发顺序

### 第一批

- 任务 5：首次诞生体验重做
- 任务 6：认主完成反馈强化
- 任务 7：进食 / 等待 / 拍准 / 拍偏 / 错过差异强化

原因：

- 这三项最直接决定 `v1.1.x` 是否真的更打动人

### 第二批

- 任务 1：转化结果解释性收口
- 任务 2：代谢边界回归
- 任务 8：陪伴感保护

原因：

- 这三项负责防止情绪层优化把底层主循环和陪伴感做坏

### 第三批

- 任务 9：状态文案人格化
- 任务 10：首次上手说明温柔化
- 任务 4：状态结果解释性展示
- 任务 3：平衡底线检查

原因：

- 这些更适合作为本版收尾和收口项

## 四、最小交付面

如果 `v1.1.x` 只能先做最小交付面，建议最少完成：

- 任务 5
- 任务 6
- 任务 7
- 任务 2
- 任务 8

这是 `v1.1.x` 最小可成立版本。

## 五、延期原则

如果本版资源不够，优先延期这些任务：

- 任务 4：状态结果解释性展示
- 任务 9：状态文案人格化
- 任务 10：首次上手说明温柔化

不建议延期这些任务：

- 任务 5：首次诞生体验重做
- 任务 6：认主完成反馈强化
- 任务 7：关键情绪差异强化
- 任务 2：代谢边界回归
- 任务 8：陪伴感保护

## 当前判断

`v1.1.x` 真正的开发重点，不是“做更多内容”，而是：

- 让用户第一次更容易心里一软
- 同时保证底层 `Token -> 能量 -> 养成` 没有被做乱

只要这两点同时做到，`v1.1.x` 就会是一个真正往前迈了一步的版本。
